# แนวทางการแก้ปัญหากล้องสั่น/สะบัดเวลาสแกนธนบัตร (Nyeta Project)

## แนวทางที่ B: ตรวจจับภาพเบลอก่อนส่ง (Local Blur Detection)
* **หลักการทำงาน:** ก่อนที่จะส่งภาพเข้าเซิร์ฟเวอร์ (Groq Vision) เราจะใช้ Canvas API คำนวณความเบลอของภาพบนเครื่องมือถือเลย (เช่น การหาค่า Laplacian variance จากข้อมูล Pixel ของรูป)
* **ข้อดี:** 
  - ลดการใช้ทรัพยากร API โควต้า (ไม่ต้องส่งรูปเสียไปให้ AI)
  - ให้ Feedback ผู้ใช้ได้ทันที
* **การนำไปใช้:** ถ้าภาพเบลอเพราะกล้องสะบัด ระบบจะไม่ส่งภาพนั้นไป แต่จะแจ้งผู้ใช้ด้วยระบบ TTS (Text-to-Speech) หรือส่งเสียงเตือนสั้นๆ (Earcon) เพื่อให้ผู้ใช้รู้ว่ากำลังเล็งไม่ถูก เช่น "ภาพเบลอ กรุณาถือนิ่งๆ"

## แนวทางที่ C: ให้ AI ช่วยบอกว่าภาพเบลอหรือตกขอบ (Prompt Engineering)
* **หลักการทำงาน:** ปรับปรุง `CURRENCY_PROMPT` ในไฟล์ `src/lib/visionPrompts.js` โดยเพิ่มคำสั่งให้ AI วิเคราะห์คุณภาพของภาพด้วย
* **ตัวอย่าง Prompt ที่จะเพิ่ม:** 
  - "หากภาพเบลอจนมองไม่ออก ให้ตอบว่า BLURRY" 
  - "หากเห็นธนบัตรแค่เสี้ยวเดียวและบอกค่าไม่ได้ ให้ตอบว่า PARTIAL"
* **การนำไปใช้:** ใน `useCurrencyScanner.js` ให้ดักจับคำตอบเหล่านี้ (`BLURRY`, `PARTIAL`) แล้วแปลงเป็นการตอบกลับด้วยเสียง เช่น:
  - ตอบกลับว่า "ภาพเบลอเกินไป ถือนิ่งๆ ครับ"
  - ตอบกลับว่า "เห็นเงินไม่เต็มใบ ขยับกล้องถอยออกมานิดนึงครับ"
* **ข้อดี:** ไม่ต้องเขียนโค้ดฝั่ง Client เพิ่มเติมมาก อาศัยความฉลาดของ AI ในการช่วยตัดสินใจ
