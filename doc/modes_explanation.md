# หลักการทำงานทางเทคนิคของ 3 โหมดใน Nyeta

แอปพลิเคชัน Nyeta มีการผสมผสานระหว่าง **การประมวลผลบนเครื่อง (Client-side)** เพื่อความรวดเร็ว และ **การประมวลผลบนคลาวด์ (Cloud AI)** เพื่อความแม่นยำสูง โดยมีหลักการทำงานในแต่ละโหมดดังนี้:

## 1. โหมดผู้ช่วยเอไอ (AI Assistant)
**สถาปัตยกรรมการทำงาน: Local Object Detection + Cloud Vision AI + Speech-to-Text**
*   **การประมวลผลภาพแบบเรียลไทม์ (Local):** ระบบจะดึงภาพจากกล้องวิดีโอ (Video Stream) มาประมวลผลด้วยโมเดล Object Detection ขนาดเล็กบนเบราว์เซอร์อย่างต่อเนื่อง เพื่อหาตำแหน่งสิ่งของ (Bounding boxes) ทำให้สามารถส่งเสียงเตือนได้ทันทีโดยไม่ต้องใช้อินเทอร์เน็ต
*   **การรับคำสั่งเสียง:** เมื่อผู้ใช้พูดถาม ระบบจะใช้ Web Speech API (Speech-to-Text) แปลงเสียงพูดเป็นข้อความ (Transcript)
*   **การวิเคราะห์ขั้นสูง (Cloud):** เมื่อผู้ใช้กดถาม ระบบจะจับภาพนิ่ง ณ วินาทีนั้น แปลงเป็นข้อมูลภาพ (Base64) แล้วส่งไปพร้อมกับข้อความคำถามผ่าน API ไปยัง **Groq Vision AI (หรือโมเดล Multimodal AI)** เพื่อวิเคราะห์
*   **การตอบกลับ:** ข้อความคำตอบจาก AI จะถูกแปลงกลับเป็นเสียงพูดด้วยระบบ Text-to-Speech (TTS) ให้ผู้ใช้ฟัง

## 2. โหมดสแกนธนบัตร (Currency Scanner)
**สถาปัตยกรรมการทำงาน: Continuous Frame Sampling + Cloud Vision AI**
*   **การดึงภาพเป็นระยะ (Sampling):** ระบบไม่ได้ส่งวิดีโอไปตลอดเวลา แต่จะดึงภาพนิ่ง (Frame) จากกล้องเป็นจังหวะต่อเนื่องกัน
*   **การตรวจสอบ (Verification):** ภาพที่ดึงมาจะถูกส่งผ่าน API ไปยังโมเดล Vision AI (ผ่าน `currencyGroq.js`) ซึ่งถูกออกแบบ Prompt มาเฉพาะเจาะจงให้จัดประเภท (Classify) ว่าในภาพมีธนบัตรไทยหรือไม่ และมีมูลค่าเท่าใด
*   **การกรองผลลัพธ์:** เพื่อป้องกันความผิดพลาด ระบบจะมีการนับเฟรมและประเมินความมั่นใจ (Confidence) หากผลลัพธ์ตรงกันหลายเฟรมติดกัน ระบบสั่น (Haptic Feedback) จะทำงานและส่งเสียงแจ้งมูลค่าผ่าน TTS

## 3. โหมดอ่านเอกสาร (Document Reader)
**สถาปัตยกรรมการทำงาน: Local Edge/Contour Detection + OCR via Cloud Vision**
*   **การหาขอบเอกสาร (Local Computer Vision):** ระบบใช้การประมวลผลภาพบนเครื่อง (ผ่าน `pageEdgeDetection.js`) ตรวจจับเส้นขอบ (Edge Detection) และรูปร่าง (Contour) เพื่อหาพิกัดมุมทั้ง 4 ด้านของกระดาษในเฟรมกล้อง
*   **การประเมินรูปทรงและเสียงนำทาง:** ระบบคณิตศาสตร์จะคำนวณพื้นที่และความเอียงของสี่เหลี่ยมที่พบ หากเอียงเกินไป ระบบจะส่งเสียงนำทาง (เช่น "ขยับกล้องขึ้น") แบบเรียลไทม์
*   **การดึงข้อความ (OCR):** เมื่อมุมกล้องขนานกับกระดาษได้สัดส่วน ระบบจะจับภาพอัตโนมัติ และส่งภาพไปยัง Cloud Vision AI เพื่อทำ OCR (Optical Character Recognition) ดึงข้อความทั้งหมดออกมา
*   **การอ่านออกเสียง:** ข้อความที่สกัดได้จะถูกส่งเข้า Text-to-Speech (TTS) เพื่ออ่านให้ผู้ใช้ฟังยาวต่อเนื่อง
